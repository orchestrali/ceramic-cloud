About

The Basics

This website helps with the building and testing of a custom scheme for evaluating bellringing compositions on complib.org.

The structure of a complib music scheme spreadsheet is explained in this issue.

On this page you can build a preliminary version of a custom music scheme, test the scheme on a method (plain course) or composition, and then download your scheme as a csv.

Viewing and Editing

Clicking on one of the stage names will display the current tests for that stage.

I have started each stage out with runs of 4+ bells, given points at front and back, and a small selection of named rows. These can be deleted individually by clicking the "x" at the end of a table row, or there is a button to clear everything for the stage for a fresh start. Items in these tables can't be edited hereâ€”you can either delete items and add them again with changes, or edit the downloaded csv file.

Some columns of the complib spreadsheet cannot be edited here at all and will need to be adjusted in the downloaded file. "Summarise" is perhaps the most important of these.

The form for adding items to the scheme, along with some more specific instructions, is below all the tables.

IMPORTANT: This website has no way to save or remember your work! If the page is reloaded, everything you have added will be lost. If you want to keep your work, be sure to download the csv! Unfortunately there is not currently a way to restore work from a previous session.

Downloading Your Work

The download button will give you a csv file with everything visible in the tables under each stage, with the additional columns needed for the final version. Even stages are together at the top, then all the odd stages. Each stage will have subtotal rows created from the test categories.

When viewing/editing the csv in Excel, it may be best to import the file rather than opening it directly. That way you can set the "Mask" column to be interpreted as text (important particularly on Royal so initial "0" doesn't get removed).

Instructions

Here are explanations of the items in the form below that do not correspond exactly to columns in the final spreadsheet.

Pattern: this corresponds to the "Mask" column of the spreadsheet, with one limitation and one addition:

The field will accept whole rows, exact or with x as a wildcard, and segments of a row (types "Row" and "Mask" in the final spreadsheet). Wraps are not available. Subtotals will be automatically generated from the category field.

Parentheses can be used to indicate that the bells inside the parentheses can be in any order. For example, entering "xxxx(57)(68)" with stage 8 will result in four rows being added to the major table:

  • xxxx5768
  • xxxx7568
  • xxxx5786
  • xxxx7586

Transpose: when entering a pattern shorter than a whole row, checking this box will cause the pattern to be added with every starting bell possible for the stage. For example, entering "1234" as the pattern and checking "transpose" will automatically add all descending four-bell runs.

Front/Middle/Back: when entering a pattern shorter than a whole row, check these to indicate where in the row it will receive points.

Description: this website will automatically add the pattern/mask to the description column. This field is for any additional text you want to appear. For example, named rows in complib's default scheme have the row followed by its name in parentheses: "15263748 (Tittums)". To recreate this you would enter just "(Tittums)" in the description field.

Category: this field is how this website builds subtotal rows in the spreadsheet output. You can see what categories currently exist for each stage in the table at the top of the page.
